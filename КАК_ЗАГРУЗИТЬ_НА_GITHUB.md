# Как загрузить проект QuizByte на GitHub

## Через GitHub Desktop:

### Шаг 1: Откройте GitHub Desktop
- Запустите приложение GitHub Desktop

### Шаг 2: Добавьте локальный репозиторий
1. В меню выберите: **File** → **Add Local Repository**
2. Нажмите **Choose...**
3. Выберите папку `c:\projects`
4. Нажмите **Add repository**

### Шаг 3: Создайте первый коммит
1. В GitHub Desktop вы увидите все измененные файлы
2. Внизу в поле **Summary** введите: `Initial commit: QuizByte Android app`
3. В поле **Description** (опционально) можете добавить описание
4. Нажмите кнопку **Commit to main**

### Шаг 4: Опубликуйте на GitHub
1. После коммита появится кнопка **Publish repository**
2. Нажмите на неё
3. Введите название репозитория (например: `QuizByte` или `quizbyte-android`)
4. Выберите, будет ли репозиторий **Private** (приватный) или **Public** (публичный)
5. **НЕ ставьте галочку** на "Keep this code private" если хотите публичный репозиторий
6. Нажмите **Publish repository**

### Шаг 5: Готово! ✅
- Ваш проект будет загружен на GitHub
- Вы получите ссылку на репозиторий вида: `https://github.com/ваш-username/QuizByte`

## Важные замечания:

⚠️ **Файл `google-services.json`** содержит конфиденциальные данные Firebase.
- По умолчанию он **НЕ** игнорируется (строка закомментирована в `.gitignore`)
- Если хотите скрыть его от публичного доступа, раскомментируйте строку `# app/google-services.json` в файле `.gitignore` ПЕРЕД коммитом

---

## Если возникнут проблемы:

1. **"Repository already exists"** - проект уже добавлен в GitHub Desktop, просто откройте его
2. **"Failed to publish"** - проверьте подключение к интернету и авторизацию в GitHub Desktop
3. **Нет кнопки "Publish"** - убедитесь, что вы сделали хотя бы один коммит
